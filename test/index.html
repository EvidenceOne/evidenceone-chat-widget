Checklist de validação manual

  • ☐ Botão renderiza inline (não fixo)
  • ☐ Clicar abre o drawer em ≤300ms
  • ☐ Drawer 400px no desktop, 100% no mobile (<768px)
  • ☐ ESC fecha e retorna foco ao botão que abriu (o foco fica visível no botão trigger)
  • ☐ Backdrop click fecha e também retorna foco ao trigger
  • ☐ Tab cicla apenas dentro do drawer (inputs e botões internos); Shift+Tab faz wrap-around
  • ☐ Caso 2: show()/hide() funcionam via console; hide() emite evento eoClose
  • ☐ Caso 3 (new-session): mensagens são limpas a cada reabertura; sessão é recriada
  • ☐ Caso 4: console.error lista props faltantes
  • ☐ Caso 5: apenas --eo-button-color e --eo-button-text-color afetam o widget
  • ☐ Token lock: tentar evidenceone-chat { --eo-green: red } no host page não muda nada interno
  • ☐ DevTools: Shadow DOM ativo; document.body sem scroll quando drawer aberto
  • ☐ Mensagens do assistente suportam markdown (negrito, itálico, listas, código, tabelas, citações)
  • ☐ XSS: enviar <script>alert(1)</script> e <img src=x onerror=alert(1)> não dispara alert
  • ☐ Eventos eoReady, eoError, eoClose disparam (Caso 1); eoClose também no hide()
  • ☐ Rede offline no abrir: drawer mostra "Não foi possível conectar"
  • ☐ Fechar drawer durante streaming → Network tab mostra fetch abortado; estado interno não muda mais
  • ☐ Erro inline por mensagem: bloquear o endpoint /partner/chat no DevTools → bolha do assistente renderiza com borda vermelha e ícone "!" clicável; clicar re-envia aquela mensagem específica
  • ☐ 401 silencioso: forçar 401 no chat → primeiro envio falha invisivelmente, re-auth acontece, segunda tentativa funciona (sem mensagem de erro visível)
  • ☐ A11y: no painel Accessibility do DevTools, drawer aparece como dialog, message list como log, bolhas como article com label role
  • ☐ A11y contraste: texto cinza em fundo claro (empty state, auth-loading) passa WCAG AA (>= 4.5:1)
